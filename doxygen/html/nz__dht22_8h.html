Description

Functions for using the DHT22 1-Wire interface

Configuration

The following defines are used to configure this module, and should be placed in the projdefs.h (or similar) file. To configure the module, the required defines should be uncommented, and the rest commented out.

// *********************************************************************
// ------------ DHT22 Configuration (from nz_dht22.h) -------------
// *********************************************************************
//---- Disable WATE functions. Not required if only asynchronous functions are used
#define DHT22_DISABLE_WATE_FUNCTIONS

Usage

To use this module, the following must be done:

  • Include nz_dht22.h in the c file it is used in.
  • Add nz_dht22.c to the MPLAB project, this is the main DHT22 driver file.
  • The following additional files are required by nz_dht22.c, and must be added to the project: nz_circularBufferPwr2.c, nz_helpers.c, nz_netcruzer.c and nz_serI2C.c
  • Add "HAS_SERPORT_I2C1" to projdefs.h file.
  • In code, initialize the DHT22. Ensure to delay 300ms from power up. For example:
    delay_ms(300);
    ds2482Init();
  • In projdefs.h, do any DHT22, I2C or other configuration required. This is done by copying the configuration section from the *.h files to projdefs.h. Nothing required, defaults will work!
  • In code, call ds2482Task() repetitively in main loop
  • All DONE! Can now use DHT22 functions defined in nz_dht22.h! Some examples: lcdDisplayString(0, "\fHello\nWorld, Again!"); //Write parsed string to LCD display lcdDisplayChar(0, 'x'); //Display 'x' at current cursor position
